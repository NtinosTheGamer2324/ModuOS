Based on your symptoms, here's what's happening:

SYMPTOM: OS boots but gets stuck showing "ModuOS     New Technologies Software 2025"

ROOT CAUSE: The shell process is blocking in input() waiting for keyboard input
that never comes on your FATAL1TY UEFI-CSM system.

DIAGNOSIS:
1. Kernel boots successfully (you see the logo)
2. Shell process is created
3. Shell calls zenith4_start() -> displays banner -> calls input()
4. input() enters busy-wait loop: while(!input_ready) { hlt; }
5. On UEFI-CSM, PS/2 keyboard might not be properly initialized
6. Keyboard interrupts never fire, so input_ready never becomes true
7. Shell hangs forever, OS appears "stuck"

EVIDENCE:
- src/drivers/input/ps2/ps2.c:154 - input() function uses blocking HLT loop
- Timer interrupts ARE firing (otherwise system would completely freeze)
- But scheduler can't help because shell process never yields
- No timeout or fallback mechanism

LIKELY CAUSES ON YOUR HARDWARE:
1. UEFI-CSM mode has different PS/2 controller initialization
2. USB legacy keyboard emulation might be interfering
3. PS/2 controller might need additional setup for UEFI systems
4. IRQ1 (keyboard) might not be properly routed in UEFI mode
